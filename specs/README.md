# Legacy specs migration notice

`specs/` 不再是 BitCal 的活动规范源。

从本次终局治理开始：

- 所有新的 proposal、design、tasks 与 requirement 变更都只写入 `openspec/`
- 产品、API、RFC、测试规范的权威版本只保留在 `openspec/specs/`
- `specs/` 仅作为历史路径的迁移壳层，帮助旧链接和旧说明找到新的规范位置

请改用以下路径：

- `openspec/specs/product/`
- `openspec/specs/api/`
- `openspec/specs/rfc/`
- `openspec/specs/testing/`
