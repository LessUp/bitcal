# Release Notes — v2.1.0

**Release Date:** February 27, 2026  
**Codename:** "Velocity"  
**Status:** Stable, recommended version

---

## 🚀 Highlights

- **New Operation:** Native ANDNOT (a & ~b) across all SIMD backends
- **Performance:** Up to 2.3× faster `is_zero()`, 1.7× faster `~`, 1.4× faster `reverse()`
- **Testing:** Full bit1024 coverage
- **Infrastructure:** ARM32 CI support, CMake flag auto-detection

---

## ✨ New Features

### ANDNOT Operation

New `andnot()` method using native SIMD instructions for `a & ~b`:

| Platform | Instruction | Performance |
|----------|-------------|-------------|
| SSE2 | `_mm_andnot_si128` | ~2.1ns (128-bit) |
| AVX2 | `_mm256_andnot_si256` | ~2.1ns (256-bit) |
| NEON | `vbicq_u64` | ~3.2ns (128-bit) |

```cpp
bitcal::bit256 a(0xFFFFFFFF);
bitcal::bit256 b(0x00FF00FF);

auto result = a.andnot(b);  // ~2× faster than (a & ~b)
// result = 0xFF00FF00
```

### Unified Binary Operation Dispatch

Refactored internals use `dispatch_binop<Op>` template:
- Reduces code duplication by ~40%
- Easier to add new operations
- Same performance (inlined)

## ⚡ Performance Improvements

### NOT Operator (~)

Previously fell back to scalar; now uses SIMD XOR with all-ones:

| Type | v2.0 | v2.1 | Improvement |
|------|------|------|-------------|
| bit128 | 2.8ns | 1.5ns | **1.9×** |
| bit256 | 3.5ns | 2.1ns | **1.7×** |
| bit512 | 7.2ns | 4.2ns | **1.7×** |

**Implementation:**
- SSE: `_mm_xor_si128` with `0xFFFFFFFFFFFFFFFF`
- AVX: `_mm256_xor_si256` with all-ones
- NEON: `veorq_u64` with all-ones

### is_zero() Detection

| Type | v2.0 | v2.1 | Improvement |
|------|------|------|-------------|
| bit128 | 2.1ns | 1.2ns | **1.8×** |
| bit256 | 4.2ns | 1.8ns | **2.3×** |
| bit512 | 8.5ns | 3.6ns | **2.4×** |

**Implementation:**
- SSE: Single `_mm_movemask_epi8` + comparison
- AVX: Single `_mm256_testz_si256` instruction
- NEON: Lane comparison with reduction

### clear() Reset

Now uses `std::memset` for better compiler optimization opportunities.

| Type | v2.0 | v2.1 | Improvement |
|------|------|------|-------------|
| bit1024 | ~15ns | ~8ns | **1.9×** (L1 cache)

### reverse() Bit Reversal

Eliminated temporary array allocation by operating in-place:

| Type | v2.0 | v2.1 | Improvement |
|------|------|------|-------------|
| bit256 | 12.1ns | 8.5ns | **1.4×** |
| bit512 | 24.5ns | 16.8ns | **1.5×** |

## 🔧 Bug Fixes

### Compiler / Platform Fixes

| Issue | Version | Fix |
|-------|---------|-----|
| MSVC SSE2 detection | v2.0.x | Added `_M_X64` check + `<intrin.h>` include |
| NEON NOT operation | v2.0.x | `vmvnq_u64` → `veorq_u64` (compatibility) |
| NEON variable shift | v2.0.x | `vshlq_n_u64` → `vshlq_u64` (runtime shifts) |
| Intrinsics scope | v2.0.x | Fixed include order (must be global scope) |

### Shift Operation Fixes

| Issue | Symptom | Fix |
|-------|---------|-----|
| SSE 256-bit shift | 4-qword carry chain breakage | Fixed carry propagation logic |
| AVX 256/512-bit shift | Cross-lane carry error | Corrected `_mm256_permute2x128` usage |
